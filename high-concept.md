# High Concept (1 Page)

## Title
Reborn in Limbo

## Genre
2D pixel-art roguelike (action-lite) with dialogue-driven memory fragments.

## Core Fantasy
Fight through Limbo while haunting memory fragments interrupt reality, forcing choices that shape your karma and determine how you rebuild yourself between runs.

## Core Loop
1. Start a Limbo run.
2. Explore/fight.
3. Find a memory fragment pickup (mostly between rooms): choose when to collect it.
4. On pickup, dialogue opens: choose how you respond.
5. Choices grant karma (good or bad) and reinforce the theme.
6. Clear combat rooms and push down the hallway chain.
7. Enter the final Boss Room; the gate locks until the boss is defeated.
8. Spend karma in the hub for permanent upgrades.
9. Restart stronger; repeat until you break the curse.

## Player Experience
Fast runs, readable combat, and emotionally loaded choices delivered as intentional “breathers” between fights (player-triggered pickups, not timed interruptions). The tone is eerie but reflective; the UI and pixel art reinforce a fractured-memory aesthetic.

## Visual Style
2D pixel art with a limited palette:
- Limbo: desaturated fog/dither, high contrast silhouettes.
- Memories: warmer tint, vignette/glitch edges, card-like overlays.

## Key Mechanics
- Permadeath runs (quick sessions)
- Memory fragment pickups that open dialogue overlays (2-3 choices) only when collected
- Split karma as progression currency (good vs bad)
- Optional breather power-ups using run karma:
  - Red circle: spend bad karma for red-orb shots
  - Gold circle: spend good karma for a timed damage ring
- Hub shop for upgrades between runs
- Final boss room encounter as a clear end goal

## AI Integration (Evidence Plan)
- AI-assisted code generation for state machine, UI overlays, and data-driven memory fragments.
- AI-assisted art generation (sprites/UI concepts) + manual pixel cleanup for consistency.
- AI-assisted debugging and balancing (document prompts + outcomes in `refinements-changes.md`).
